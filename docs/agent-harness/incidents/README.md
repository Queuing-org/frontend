# Queuing Incidents and Improvements

Durable troubleshooting, performance, and reusable-fix records live here when the work changes how future work should be done.

Use `docs/portfolio-notes/` for scratch notes, performance-improvement drafts, and portfolio case-study material. Promote only confirmed or explicitly useful lessons into this folder.

## Naming

```text
YYYY-MM-DD-short-topic.md
```

Example:

```text
2026-05-03-room-update-patch-payload.md
```

## Required Sections

- Problem
- Previous Behavior
- Previous Code
- Updated Code
- Problem in the Previous Code
- Evidence
- Cause or Remaining Hypotheses
- Solution Options
- Chosen Solution and Rationale
- Result
- Reusable Rule
- Skill or Team Spec Updates
- Verification
