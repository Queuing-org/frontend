# Product Specifications

Create a product spec when behavior spans multiple screens, roles, or backend events and cannot be recovered safely from one implementation file.

Each spec should define:

- user goal and supported roles
- normal, loading, empty, error, and permission-denied behavior
- API and real-time dependencies
- cross-screen consistency requirements
- acceptance scenarios

The root [`README.md`](../../README.md) remains the current high-level feature index. Add focused specs here as behavior is clarified or changed.
