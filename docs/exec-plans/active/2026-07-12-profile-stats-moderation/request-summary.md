# Request Summary

프로필의 더미 통계를 실제 `musicPower`, `queuingCount` 데이터로 바꾸고 이용 시간을 제거한다. 음악력 조회/추천을 추가하며 본인/비로그인/이미 추천한 사용자에 대한 버튼 상태를 제한한다. 채팅 행에는 작성자 및 메시지 식별자에 따라 신고/차단 관리 메뉴를 제공하고, 접근 가능한 확인/선택 모달과 API mutation/cache 처리를 구현한다. Vitest와 Testing Library를 도입하고 기능 단위 한국어 커밋 6개로 나눠 Draft PR까지 전달한다.

후속 요청으로 이용 시간은 기존 하드코딩 UI를 복구하고, 음악력 액션은 프로필 패널 우측 하단으로 이동한다. 차단한 사용자의 채팅은 성공 즉시 숨기며 새로고침 후 서버 차단 안내 메시지도 렌더링하지 않는다. PR #26의 CodeRabbit 피드백도 유효성 검토 후 반영한다.
