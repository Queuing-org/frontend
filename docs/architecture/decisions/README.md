# Architecture Decisions

Create a short decision record when a change alters a top-level layer, dependency direction, state owner, shared contract, or cross-cutting runtime policy.

Use `YYYY-MM-DD-short-slug.md` and include:

- status: proposed, accepted, superseded, or rejected
- context and constraints
- options considered
- decision and rationale
- consequences and migration work
- verification or enforcement

Small implementation choices that do not affect future work do not need a decision record.
